# Touch Tone

> *"Those dulcet tones from the before times."*

A sample [Pebble Time 2](https://repebble.com/) watch app that
plays the touch tone classic sounds on an old 12-button telephone.

No menus, no animations, no network. Just an old-school touch tone phone keypad.

This is a demo PT2 touch and speaker sound app, it's not a real dialer.

---

## Features

- **12-key DTMF keypad** rendered as the entire UI, sized and centered to the
  Pebble Time 2 (Emery) 200 × 228 display.
- **Touch-to-play tones** Touch the on-screen buttons to play the
  authentic North American DTMF (Dual-Tone Multi-Frequency) tone pair for as
  long as a button is held.
- **Dial tone** On the Select button, played for as long as the button is held.
- **Busy signal** On the Up button, looping for as long as the button is held.
- **Special Information Tone** (SIT "your call cannot be completed" error tone)
  played once on the Down button.

## How it works

Every sound is raw PCM streamed to the speaker with `speaker_stream_open()` /
`speaker_stream_write()`, rather than `speaker_play_tone()` (which has a known
noise issue) or `speaker_play_notes()` (MIDI-only — and DTMF frequencies are not
MIDI notes). A timer keeps a small lead of audio (~50 ms) queued ahead of
playback so the stream doesn't underrun; that lead is also the onset latency at
the start of a sound. On release the app calls `speaker_stop()` (not
`speaker_stream_close()`, which would play out the queued audio) so the sound
ends immediately.

The **keypad tones are synthesized in real time**: two phase accumulators step
through a 256-entry sine table and their outputs are summed each sample. Because
the phase runs continuously, there are no buffer-loop boundaries — and so none of
the ~10 Hz pulsing that a repeated, non-harmonic 800-sample buffer produced.

The **other sounds use pre-computed 16-bit/8 kHz buffers** (generated offline by
`generate_audio.py`): the dial tone loops one two-tone buffer, the busy signal
alternates a tone buffer and a silence buffer for its 0.5 s on/off cadence, and
the SIT steps through its three single-frequency tones and gaps by exact sample
count, then closes the stream so it ends on its own.

## Building

Built with the Pebble **C SDK** (`sdkVersion 3`).

Requires the [Pebble SDK](https://github.com/coredevices/pebble-tool)
(`pebble` CLI). From the project root:

```bash
# (Re)generate the audio buffers — only needed if you change tones.
pip install numpy
python3 tools/generate_audio.py

# Build and run in the local emulator.
pebble build
pebble install --emulator emery
pebble logs --emulator emery
```

> **Audio note:** the browser/CloudPebble emulator does not play audio, and the
> local emulator's audio depends on your host configuration. Treat real Pebble
> Time 2 hardware as the definitive audio test target.

### On-device

```bash
# In the Pebble mobile app: Settings → Connectivity → Use LAN developer connection. Then, Devices → ⋯ → Enable Dev Connect → note the IP.
pebble install --phone <IP_ADDRESS>
pebble logs --phone <IP_ADDRESS>
```

## Regenerating audio

`src/c/touch_tone_audio.h` is generated by `tools/generate_audio.py` (it is
committed so the project builds without numpy). To change tone frequencies,
amplitude, or buffer length, edit the script and re-run it — never hand-edit the
header.

## Controls

| Input | Action |
|---|---|
| Tap a key | Play that key's DTMF tone while held |
| Select | Play the dial tone while held |
| Up | Play the busy signal (looping) while held |
| Down | Play the Special Information Tone (SIT) once |
| Back | Exit the app |
